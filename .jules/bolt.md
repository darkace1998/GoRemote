## 2026-06-20 - Rejecting symlinked files securely
**Learning:** When preventing path traversal via symlinks, the safest and simplest approach is often to outright reject non-regular files (using `!lstatInfo.Mode().IsRegular()`) rather than attempting complex symlink resolution and path math (`filepath.EvalSymlinks` and `filepath.Rel`). Additionally, when rejecting plugins during discovery in this codebase, explicitly registering an entry with `StatusBroken` and an error message is preferred over silently skipping the directory with `continue`.
**Action:** When validating file types during traversal or discovery, enforce regular files (`IsRegular()`) immediately. For this specific registry system, ensure invalid entries are recorded as `StatusBroken` instead of being omitted from the registry state to provide clear feedback.
