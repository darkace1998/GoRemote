# Plugin IPC transport

`host/plugin/ipc` implements the reference transport between the goremote
host and out-of-process plugins, as outlined in `architecture.md`.

## Wire format

- **Transport**: gRPC over a Unix domain socket (POSIX). Windows
  named-pipe support is planned but not yet implemented; the package
  compiles on Windows but every transport call returns an explicit
  "not yet implemented" error.
- **Codec**: a hand-rolled protobuf codec registered as
  `goremote-pluginv1`. It exists because the goremote sandbox does not
  always have `protoc` available; once it does, the codec can be
  removed and the messages regenerated by `protoc-gen-go` /
  `protoc-gen-go-grpc` without breaking the public API.

## Services (`proto/plugin/v1/plugin.proto`)

- `PluginHandshake.Hello` — first call after connect; the plugin
  declares its id, version, capabilities, and a status string of
  `ready` or `degraded:<reason>`.
- `Echo.Ping` — smoke test / liveness probe; the plugin echoes the
  request payload.

Higher-level protocol and credential RPCs will layer on top of the
same connection; only the contract above is part of the v1 surface.

## Build-tag policy

Platform-specific socket primitives live in two files guarded by build
tags:

| File                     | Build tag        | Purpose                                          |
|--------------------------|------------------|--------------------------------------------------|
| `socket_unix.go`         | `//go:build unix`    | Real `net.Listen("unix", …)` implementation. |
| `socket_windows.go`      | `//go:build windows` | Stub returning `errWindowsUnsupported`.       |

Tests that exercise a live socket (`ipc_test.go` and the
`plugins/external-example` e2e test) carry `//go:build unix`. The
public package API (`ListenUnix`, `DialUnix`, `Server`, `Client`)
compiles on every platform; only runtime behavior differs.

## Listener semantics

- `ListenUnix` removes a stale socket file but refuses
  (`ErrSocketInUse`) if an active process is accepting on it.
- The socket is `chmod 0600` so only the owning user can connect.
- `Listener.Close` and `Server.Stop` both unlink the socket file.

## Server lifecycle

`Server.Serve(ctx)` blocks until `ctx` is cancelled. On cancel it issues
a `GracefulStop` with a 5-second budget; if that does not complete, it
falls back to `Stop`. The socket file is always cleaned up before
`Serve` returns.

## Client dial

`DialUnix` waits for the gRPC connection to enter `READY` (default
timeout 5 seconds, override with `WithDialTimeout`) so a missing or
unresponsive socket fails fast at dial time rather than on the first
RPC.
